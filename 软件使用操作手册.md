# Multi-Platform Novel Auto Publish

This workspace now contains both:

- `qidian`: Qidian writer backend automation with unattended/scheduled mode.
- `fanqie`: The complete working Fanqie implementation imported from `D:\code\fanqie_auto_publish`.
- `faloo`: Faloo writer backend automation via `https://author.faloo.com/OpusList2020.aspx`.
- `qimao`: Qimao writer backend automation via `https://zuozhe.qimao.com/`.

Platform registry lives in `platform_config.py`.

## Login

Qidian:

```bash
py login.py qidian
```

Fanqie:

```bash
py login.py fanqie
```

Faloo:

```bash
py login.py faloo
```

Qimao:

```bash
py login.py qimao
```

State files are separated by platform:

- `state_qidian.json`
- `state_fanqie.json`
- `state_faloo.json`
- `state_qimao.json`

Existing `state.json` remains a fallback for compatibility.

## Qidian Publish

Interactive:

```bash
py publish.py --platform qidian
```

Unattended/scheduled:

```bash
py publish.py --platform qidian --no-prompt --headless
```

Specific book/count:

```bash
py publish.py --platform qidian --book "BookName" --count 3 --no-prompt --headless
```

Windows Task Scheduler can run:

```text
D:\code\qidian_auto_publish\run_unattended_qidian.bat
```

Logs are appended to `logs\scheduled_publish.log`.

## Fanqie Publish

The Fanqie code is imported as a platform-specific original implementation under `platforms/`.

Login:

```bash
py login.py fanqie
```

Command-line publish:

```bash
py publish.py --platform fanqie
```

Fanqie GUI:

```bash
py run_fanqie_gui.py
```

Note: the imported Fanqie implementation is currently interactive, matching the original working project. It is intentionally not forced into `--no-prompt --headless` until its manual-assistance points are adapted safely.

## Faloo Publish

Login:

```bash
py login.py faloo
```

Interactive publish:

```bash
py publish.py --platform faloo
```

Unattended publish after login state is saved:

```bash
py publish.py --platform faloo --book "BookName" --count 1 --no-prompt --headless
```

Faloo uses the author work list at `https://author.faloo.com/OpusList2020.aspx` and stores cookies in `state_faloo.json`. If the backend changes button text or requires a manual confirmation, the script saves screenshots and HTML under `debug/`.

## Qimao Publish

Login:

```bash
py login.py qimao
```

Interactive publish:

```bash
py publish.py --platform qimao
```

Unattended publish after login state is saved:

```bash
py publish.py --platform qimao --book "BookName" --count 1 --no-prompt --headless
```

Qimao uses `https://zuozhe.qimao.com/` and stores cookies in `state_qimao.json`. The first run may need manual inspection because Qimao is a single-page app and route names can change.

## Folder Rules

Use one platform folder, then one folder per book:

```text
chapters/
  qidian/
    QidianBookName/
      001 Chapter 1 Title.txt
  fanqie/
    FanqieBookName/
      001 Chapter 1 Title.txt
  faloo/
    FalooBookName/
      001 Chapter 1 Title.txt
  qimao/
    QimaoBookName/
      001 Chapter 1 Title.txt
```

Published files are moved to the matching platform archive folder:

```text
uploaded/
  qidian/
  fanqie/
  faloo/
  qimao/
```

For compatibility, if `chapters/<platform>/` does not exist but the old shared `chapters/<book-name>/` folders contain txt files, command-line publishing falls back to the old layout and prints a migration warning. New files should use the platform-isolated layout.
